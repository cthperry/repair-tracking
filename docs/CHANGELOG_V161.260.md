# CHANGELOG｜V161.260

日期：2026-02-12（Asia/Taipei）

## Phase 4（優化）
### Dashboard / KPI
- 新增 KPI：
  - **本週 MTTR**：本週「完成」維修單的平均處理天數（createdAt → completedAt）。
  - **本週轉單率**：本週「報價非草稿」數量為分母、訂單新建數為分子（同週）計算百分比。
- KPI 卡片分為「可點擊導頁」與「純顯示」兩類：新增的 MTTR / 轉單率為純顯示（不導頁），避免誤觸。
- 本週摘要列補充顯示 MTTR / 轉單率，同時保留既有「平均處理天數」欄位。

### 通知中心（未下單提醒強化）
- 「需收費但尚未下單」提醒邏輯優化：
  - `orderStatus = 'not_ordered'` 且已填 **未下單原因** → 視為已結案追蹤，不再重複提醒。
  - `orderStatus = 'not_ordered'` 但未填原因 → 仍提醒（提示補齊原因/備註）。
  - 提醒文字會帶出原因與備註（若有），便於快速判讀。
  - `decidedAt` 優先作為計算基準，避免因更新其他欄位造成提醒天數跳動。
