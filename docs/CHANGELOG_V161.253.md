# CHANGELOG｜RepairTracking V161.253（2026-02-11｜Asia/Taipei）

## 新增
- 維修單新增「收費/下單」欄位：
  - 維修後是否需要收費（尚未決定 / 不需收費 / 需收費）
  - 客戶是否下單（尚未確認 / 已下單 / 未下單）
  - 未下單原因（價格過高 / 客戶預算不足 / 客戶內部流程/延後 / 其他）
- Dashboard 通知新增：需收費且 7 天以上仍未下單/未確認 → 提醒追蹤。
- Analytics 新增「收費/轉單」統計卡：
  - 需收費/不需收費/未決定數量
  - 需收費 → 已下單轉單率
  - 未下單原因彙總
- 週報新增每筆維修單的「收費/下單狀態」行。

## 自動同步
- 建立訂單（從報價/維修單轉訂單）後，會同步回寫維修單：
  - 收費：需收費
  - 下單：已下單

## 相容與規則
- Phase 1 DoD 不變：Service 存取仍只允許 AppRegistry / window._svc。
- 資料承載採用 repair.billing 物件（避免新增大量頂層欄位），RepairModel.update 已支援 billing 深合併。
