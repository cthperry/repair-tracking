# RepairTracking V161.181 變更紀錄

日期：2026-01-11

## 修正

### 訂單/採購追蹤列表顯示為空白（但上方顯示有筆數）

**現象**
- 「訂單/採購追蹤」頁面標題列顯示「共 N 筆」，但列表區塊顯示「目前沒有資料」。

**根因**
- 訂單列表的進階篩選中，「金額範圍」的數值解析函式會把空字串 `''` 視為數值 `0`。
- 因此在未輸入任何金額範圍時，仍會套用 `totalAmount <= 0` 的篩選，導致多數訂單被過濾掉，列表看似空白。

**修正**
- 空字串不再被解析為 0；空值會視為 `null`（不套用金額範圍篩選）。

## 影響範圍
- 僅修正 UI 層的數值解析，不變更 Firebase 資料結構與既有訂單資料。

## 變更檔案
- `features/orders/orders.ui.js`
- `core/config.js`
